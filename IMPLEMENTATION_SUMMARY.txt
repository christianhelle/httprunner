INSECURE HTTPS SUPPORT - IMPLEMENTATION SUMMARY
================================================

Problem Statement:
------------------
Implement support for insecure HTTPS connections in httprunner to allow testing
against endpoints with self-signed or invalid SSL certificates. Since Zig's 
standard library HTTP client doesn't support this, switch to libcurl.

Solution Overview:
------------------
Successfully implemented end-to-end insecure HTTPS support using libcurl as the
HTTP client backend. The implementation allows users to bypass SSL certificate
verification via both command-line flags and per-request directives.

Usage:
------

Command-line (global):
  httprunner myfile.http --insecure
  httprunner myfile.http -k

Per-request directive:
  # INSECURE
  GET https://self-signed.badssl.com/
  
  INSECURE
  POST https://expired.badssl.com/api/test

Testing:
--------
- Works with self-signed certificates
- Works with expired certificates  
- Works with untrusted root CAs
- Works with hostname mismatches
- Example file: examples/insecure.http

Dependencies:
-------------
Build: libcurl development libraries
Runtime: libcurl shared library

Platform installation:
  Ubuntu/Debian: sudo apt-get install libcurl4-openssl-dev
  Fedora/RHEL: sudo dnf install libcurl-devel
  macOS: brew install curl

Security:
---------
⚠️ WARNING: Only use insecure mode in development/testing environments
⚠️ Never use with production endpoints or sensitive data
⚠️ Disabling SSL verification enables man-in-the-middle attacks
